Modeling life expectancy with lifeplus

library(lifeplus)
library(ggplot2)

Introduction

The lifeplus package allows you to fit a variety of Bayesian hierarchical models to life expectancy data. The models are modular, allowing you to mix and match multiple types of model components (Susmann, Alexander, and Alkema 2022).

The model

Let \(c = 1, \dots, C\) index the areas (typically countries) and \(t = 1, \dots, T\) index time points (typically years). Let \(y_{c,t}\) be the observed life expectancy at birth in area \(c\) at time \(t\). The core of the model is the following equation:

\[ y_{c,t} = y_{c,t-1} + g\left(y_{c,t-1}, \beta_c\right) + \delta_{c,t} + \epsilon_{c,t} \]

where

  • \(g(\cdot)\) is a transition function that models the expected rate of change of the outcome as a function of its previous level, using area-specific parameters \(\beta_c\). This term captures long-term, systematic trends in life expectancy.
  • \(\delta_{c,t}\) is a shock term representing large departures from the overall trend.
  • \(\epsilon_{c,t}\) is a residual capturing short-term departures from the overall trend, which we model with a data model.

Each of these three components can be configured independently. For full details on the available options, see the documentation for each constructor (e.g, ?transition_model_double_logistic).

Transition models

  • Double logistic: transition_model_double_logistic. The transition is modeled by a parametric double logistic function (Raftery et al. 2013).
  • Spline: transition_model_spline. The transition is modeled by a B-spline function.
  • Gaussian Process: transition_model_gaussian_process. The transition function is assigned a Gaussian Process prior, approximated using a basis function expansion method (Riutort-Mayol et al. 2023).

All transition models support a hierarchical option that enables partial pooling of transition parameters across areas.

Shock models

  • None: shock_model_none. All shock terms are fixed to zero: \(\delta_{c,t} = 0\).
  • Regularized horseshoe: shock_model_regularized_horseshoe. Sparse shrinkage prior that allows a small number of large shocks (Piironen and Vehtari 2017).

Data models

  • Normal: data_model_normal. The residuals are modeled as \(\epsilon_{c,t} \sim N\left(0, \sigma^2\right)\).
  • Outlier: data_model_outlier. Observations with absolute differences exceeding a threshold are excluded; remaining residuals are normal.

World Population Prospect Data

We use male life expectancy estimates from the United Nations World Population Prospects 2024, available through the wpp2024 R package.

We’ll work with Bosnia and Herzegovina as an initial example.

library(wpp2024)
data(UNlocations, package = "wpp2024")
data(e0M1, package = "wpp2024")

bosnia_herzegovina_codes <- UNlocations |>
  dplyr::filter(name == "Bosnia and Herzegovina") |>
  dplyr::pull(country_code)

bosnia_herzegovina <- e0M1 |>
  dplyr::filter(country_code %in% bosnia_herzegovina_codes) |>
  tidyr::pivot_longer(
    cols = `1950`:`2023`, 
    names_to = "year", 
    values_to = "e0"
  ) |>
  dplyr::mutate(year = as.numeric(year))
ggplot(data, aes(x = year, y = e0)) +
  geom_point() +
  labs(
    x = "Year", 
    y = "Male life expectancy at birth (e0)", 
    title = "Bosnia and Herzegovina"
  )

Fitting a basic model

Before fitting models, let’s define a wrapper with shared sampling settings. This helps us keep the examples concise, and ensures we use the same MCMC settings across all the models we fit:

my_lifeplus <- function(data, ...) {
  lifeplus(
    data = data, 
    y = "e0", 
    time = "year", 
    area = "name", 
    end_time = 2050, 
    iter_warmup = 1e3,
    iter_sampling = 1e3,
    refresh = 0,
    show_messages = FALSE,
    show_exceptions = FALSE,
    parallel_chains = 4,
    adapt_delta = 0.99,
    ...
  )
}

We start with a simple configuration: a double logistic transition model, normal data model, and no shock terms. As we’re fitting a single country, hierarchical estimation across areas is not applicable, so we set hierarchical = FALSE:

fit_basic <- my_lifeplus(
  data = bosnia_herzegovina,
  transition_model = transition_model_double_logistic(hierarchical = FALSE),
  data_model = data_model_normal(), 
  shock_model = shock_model_none()
)

The plot_projections function shows the posterior projections overlaid on the observed data:

plot_projections(fit_basic, data = TRUE)

The plot_transitions function shows the estimated transition function (the expected rate of change in male life expectancy as a function of its level):

plot_transitions(fit_basic)

Adding shock terms

By adding a regularized horseshoe shock model, we can allow the model attribute large deviations in life expectancy to specific country-years, allowing the underlying transition model to capture longer-term smooth trends.

fit_shocks <- my_lifeplus(
  data = bosnia_herzegovina, 
  transition_model = transition_model_double_logistic(hierarchical = FALSE),
  data_model = data_model_normal(), 
  shock_model = shock_model_regularized_horseshoe(scale_global = 0.1)
)
#> Extracting posteriors...
plot_projections(fit_shocks)

The estimated shock terms reveal when the model detected large short-term deviations from the expected trend:

plot_shocks(fit_shocks)
#> Warning in ggplot2::geom_errorbar(ggplot2::aes(ymin = `2.5%`, ymax = `97.5%`, : Ignoring unknown aesthetics: fill

The transition function now better captures the shape of the data, and exhibits tighter uncertainty estimates than the basic model without shocks:

plot_transitions(fit_shocks)

Validating models

We can assess predictive performance by holding out recent data. Setting cutoff_time = 2015 fits the model using only data up to 2015. We can then evaluate how well the projections match the observed 2016-2023 values:

fit_valid_basic <- my_lifeplus(
  data = bosnia_herzegovina,
  transition_model = transition_model_double_logistic(hierarchical = FALSE),
  data_model = data_model_normal(), 
  shock_model = shock_model_none(),
  cutoff_time = 2015
)
#> Extracting posteriors...

fit_valid_shocks <- my_lifeplus(
  data = bosnia_herzegovina,
  transition_model = transition_model_double_logistic(hierarchical = FALSE),
  data_model = data_model_normal(), 
  shock_model = shock_model_regularized_horseshoe(scale_global = 0.1),
  cutoff_time = 2015
)
#> Extracting posteriors...

The projection plots automatically distinguish the held-out observations:

plot_projections(fit_valid_basic, data = TRUE) + ggtitle("Basic")

plot_projections(fit_valid_shocks, data = TRUE) + ggtitle("Shocks")

The posterior median of the residual standard deviation parameter (\sigma_\epsilon) is about 10 times higher in the basic model than the shocks model. Without the shock term, the basic model has to use the residuals to model the large changes in life expectancy, which subsequently drives up the variance of the projections.

Hierarchical estimation with multiple countries

Hierarchical estimation allows for the transition functions to borrow information across countries. To illustrate, let’s fit a model to multiple East African countries.

east_africa_names <- c("Somalia", "Ethiopia", "Kenya", "Djibouti")
east_africa_codes <- UNlocations |>
  dplyr::filter(name %in% east_africa_names) |>
  dplyr::pull(country_code)

east_africa <- e0M1 |>
  dplyr::filter(country_code %in% east_africa_codes) |>
  tidyr::pivot_longer(
    cols = `1950`:`2023`,
    names_to = "year",
    values_to = "e0"
  ) |>
  dplyr::mutate(year = as.numeric(year))
ggplot(east_africa, aes(x = year, y = e0, color = name)) +
  geom_point() +
  labs(
    x = "Year",
    y = "Male life expectancy at birth",
    title = "East Africa"
  )

Next, fit the model with hierarchical = TRUE to estimate the double logistic parameters hierarchically across countries:

fit_hier <- my_lifeplus(
  data = east_africa,
  transition_model = transition_model_double_logistic(hierarchical = TRUE),
  data_model = data_model_normal(),
  shock_model = shock_model_regularized_horseshoe(scale_global = 0.1)
)

dplyr::bind_rows(
  tidy(fit_hier) |> dplyr::mutate(model = "hierarchical"),
  tidy(fit_indep) |> dplyr::mutate(model = "independent")
) |>
  ggplot(aes(x = time, y = estimate, color = model)) +
  geom_line() +
  facet_wrap(~area)
